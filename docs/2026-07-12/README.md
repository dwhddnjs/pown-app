## 지금 당장 고칠 것 Top 5 — ✅ 전부 수정 완료 (2026-07-12)

1. ✅ **`npx tsc --noEmit`이 아예 안 돌아간다** — `typescript@5.3.3`이 Expo SDK 53의 `"module": "preserve"` 옵션(TS 5.4+ 필요)과 충돌.
   → **완료**: `typescript ~5.8.3`으로 업그레이드, `npx tsc --noEmit` 에러 0개 통과 확인. → [04 #1](./04-dependencies-config.md)
2. ✅ **세트 ID 중복 버그** — 세트를 지웠다 추가하면 ID가 겹쳐, "완료" 버튼 하나가 세트 두 개를 동시에 완료 처리한다.
   → **완료**: `Date.now()` ID로 교체 (세트·사진·숏츠 3곳 + edit-plan의 index 재부여 제거). → [01 #2](./01-bugs.md)
3. ✅ **"전체노트 열기"를 누르는 순간 퀵노트/기존 노트 내용이 삭제된다** — 수정 화면에서 특히 데이터 유실.
   → **완료**: content 삭제 로직 제거 + 노트 모달에 기존 값 프리필 + 모달 닫힐 때 임시 스토어 정리. → [01 #3](./01-bugs.md)
4. ✅ **데이터 초기화 버튼에 확인 절차가 없다** — 원터치로 모든 기록이 날아간다. 복원(JSON 가져오기)도 검증 없이 스토어에 그대로 주입된다.
   → **완료**: 확인 다이얼로그 + 복원 파일 스키마 검증 + 실패 시 에러 토스트(console 제거). → [01 #11](./01-bugs.md)
5. ✅ **숏츠 영상이 앱 캐시 경로로 저장된다** — iOS가 캐시를 비우면 촬영한 영상이 재생 불가가 된다.
   → **완료**: 사진 라이브러리 asset `localUri` 저장 + 썸네일 documentDirectory 복사 (15초 미만 영상 썸네일 실패 버그도 함께 수정). → [01 #6](./01-bugs.md)

## 전반적인 인상

- 오프라인 우선 + Zustand-as-DB 구조 자체는 단순하고 잘 동작한다. 영속/임시 스토어 분리도 명확하다.
- 가장 큰 리스크는 **`app/_layout.tsx`(815줄)의 헤더 콜백 안에 저장 로직이 들어가 있는 구조**다. 같은 이미지 저장 로직이 3벌 복붙되어 있고, 화면(라우트) 파일이 아닌 레이아웃에서 폼 제출이 일어난다. → [03 #1](./03-refactoring.md)
- ID 생성 규칙("`Date.now()` 사용, 순차 ID 금지")이 CLAUDE.md에 명시돼 있음에도 **세트·숏츠·사진 3곳에서 순차 ID를 쓰고 있고**, 그중 세트는 실제 버그로 이어진다.
- 린터·테스트가 전혀 없어 위 문제들이 걸러질 장치가 없다. `eslint-config-expo` 도입만으로도 (미사용 import, rules-of-hooks, 죽은 코드) 상당수가 자동 검출된다.
