## 1. 배포 전 고쳐야 할 것 (High)

### H-1. 라이트 모드 탭바 — 비활성 아이콘/라벨이 사실상 안 보임

`app/(drawer)/(tabs)/_layout.tsx:59-82`

탭바가 `BlurView(intensity=80)` 투명 배경인데 **`tabBarInactiveTintColor`를 지정하지 않아**
RN 기본 회색이 쓰인다. 뒤에 어두운 콘텐츠(사진 썸네일)가 깔리면 블러가 어두워지면서
회색 아이콘이 배경에 묻힌다.

- 실제 확인: 라이트 모드 운동계획 탭(사진 있는 기록 스크롤 상태)에서 `기록`·`내 숏츠`
  아이콘과 라벨이 **거의 판독 불가**. 같은 탭바가 기록 탭(밝은 배경)에서는 정상.
- 함께: `tabBarLabelStyle.fontSize: 8` — iOS HIG 탭 라벨 권장(10)보다 작다. 8pt 한글은
  가독성 하한 아래다.
- 함께: 운동계획 아이콘만 `SymbolView size={32}`, 나머지는 `size={24}` → **탭바 아이콘 크기 불균일**
  (크롭에서 덤벨만 눈에 띄게 큼).

> 수정: `tabBarInactiveTintColor` 명시(라이트에서 최소 `#767680` 이상 대비), 라벨 10pt,
> 아이콘 24로 통일.

### H-2. 달력 "쉰 날" ✕ 표시가 두 테마 모두 보이지 않음

`components/calendar/calendar-grid.tsx:84-89`

전수 크롭으로 확인했다. 운동 안 한 날 표시가 사실상 빈 원과 구별되지 않는다.
`themeColor.subText`(다크 #9a9aa0 / 라이트 #767680)를 쓰면 즉시 해결된다.

### H-3. 세트 "완료" 버튼 텍스트 대비 미달

`components/workout-plan/set-list-item.tsx:94-105`

배경을 `tint`로 올리거나 텍스트를 `themeColor.background`로 반전할 것.

### H-4. 라이트 모드 전반 — teal(`#00BAAB`) 텍스트 대비 2.4:1

`constants/Colors.ts` light `tint: "#00BAAB"`

흰 카드/`#EFEEF3` 배경 위 teal 텍스트가 전부 **약 2.4:1**로 AA(4.5:1)는 물론
큰 글씨 기준(3:1)도 못 넘는다. 해당되는 곳이 매우 넓다:

- 기록 탭 월간 요약 숫자(`7`, `2일`, `1.0`, `14%`), `가슴 (6회)`, `달력으로 보기`

- add-plan / add-multi-plan **운동 칩 텍스트 전부**, 기구 종류 세그먼트 미선택 라벨

> 채워진 배경(teal 버튼)에는 지금 색을 그대로 써도 된다.

## 2. 간격·정렬 통일성 (Medium)

### M-1. `flexWrap` + `justifyContent:"center"` 때문에 마지막 줄이 가운데로 밀림

같은 패턴이 세 군데에서 **좌측 정렬선을 무너뜨린다.**

| 위치 | 증상 |
|---|---|
| `components/chart/condition-count.tsx:90-97` | 컨디션 별 횟수 9개 중 1행 5개 / 2행 4개인데 **2행이 가운데 정렬**되어 열이 어긋난다(1행 중심 224·413·601·790·979 vs 2행 319·508·697·886 — 정확히 반칸 밀림) |
| add-plan 운동 칩 | 행마다 시작 x가 104 / 80 / 52 / 424 로 **좌측 들쭉날쭉**, 마지막 줄(`딥스`·`풀오버`)만 가운데 |
| add-multi-plan 운동 칩 | 동일. 행 시작 x 85 / 68 / 113 |

### M-2. 세트 행 구분선 / 진행바 좌우 기준선이 서로 다름

`components/workout-plan/set-list-item.tsx:132-140`

- 세트 행 하단 구분선: x 390 → 952

`sbd-chart.tsx:86`)

### M-4. 카드 제목만 `marginLeft: 6` 하드코딩

`월간 요약`·`컨디션 별 횟수`·`주로 뭐로 운동했지?`·`3대중량의 변화` 전부
`<Text style={{ fontSize: 18, marginLeft: 6 }}>`. 카드 패딩(12)에 6을 더한 값이라
**제목만 구분선·차트보다 6px 안쪽**으로 들어간다. 카드 패딩을 18로 올리고 marginLeft를
없애는 게 맞다.

### M-5. 날짜 헤더의 점이 세로 중앙에서 4px 내려가 있음

`app/(drawer)/(tabs)/workout.tsx:282-287` — `alignItems:"center"` 컨테이너 안의
`dot`에 `marginTop: 4`가 남아 있다. 크롭에서 눈에 띈다.

### M-6. select-type 모달의 세로 균형

부위 선택 화면이 상단 1/3에만 콘텐츠가 몰리고 **아래 절반 이상이 빈 검은 화면**이다.
(다크·라이트 동일) 콘텐츠를 세로 중앙에 두거나 부위 아이콘을 키우는 편이 낫다.

### M-7. 몸무게 차트만 축·격자선이 없다

`components/chart/body-chart.tsx` — 같은 기록 탭 안에서 다른 두 막대 차트는

## 3. 조작성·기타 (Medium/Low)

### M-8. 드로어 폴더 행 탭 영역이 20pt

`app/(drawer)/_layout.tsx:322-329` — `styles.header`에 세로 패딩이 없어
행 높이가 아이콘 높이(20pt)와 같다. **애플 권장 44pt의 절반 미만**이라 실제로
검수 중 정확히 조준하기 전까지 두 번 헛쳤다. `paddingVertical: 12` 정도 필요.

덧붙여 드로어 텍스트가 제목 빼고 **전부 `subText` 회색**이라 폴더 트리 전체가
비활성처럼 읽힌다. 파일/폴더 이름은 `themeColor.text`가 맞다.

### M-9. 검색창 플레이스홀더가 영어 하드코딩

`app/workout/search.tsx:66` — `placeholder: "Search"`.
바로 아래 `cancelButtonText`는 `t("search.cancel")`로 번역하는데 플레이스홀더만 빠졌다.
한국어 앱에서 `Search`가 그대로 보인다. (스크린샷 확인)

### M-10. 중량 계산기 100% 행이 라벨 없이 이모지 하나

`app/(modals)/calculate.tsx:40-42` — 110%는 `💪🏻 + 10%`, 105%는 `💪🏻 + 5%`인데
100%만 `💪🏻` 단독이라 **깨진 행처럼 보인다.** `💪🏻 1RM` 같은 라벨을 붙일 것.

같은 화면: 상단 큰 입력칸에 **어떤 값을 넣는지 라벨이 없다**(테두리도 없어 입력창인지도
불분명). 목록이 5%까지 22행 내려가는데 하위 구간은 실사용 가치가 없다.

### M-11. 컨디션 원 안 라벨이 원 테두리에 닿는다

`components/add-plan/condition-icon.tsx:134-144, 165-167` — 50pt 원 안에

5. **H-6 숏츠 그리드 하단 인셋** — 마지막 줄이 가리는 건 기능 손실

H-4(라이트 teal 대비)와 M-1(정렬)은 범위가 넓으니 다음 사이클에 묶어서 하는 편이 안전하다.

## 5. 수정 결과 (같은 날 반영 완료)

보고서의 High 6 · Medium 11 · Low 7을 수정한 뒤, **레이아웃을 바꾼 일부는 사용자 요청으로
원복**했다. `npx tsc --noEmit` 에러 0, `npm run lint` 에러 0(경고 74 → 50).
아래는 실제로 시뮬레이터에서 눈으로 재확인한 결과다.

> **원복된 항목** (레이아웃 변경분만, 대비 개선 색상은 유지)
> — 드로어(M-8 행 높이·글자색), 계산기(M-10 입력 라벨·구분선·100% 행), 내 숏츠(H-6 그리드
> 여백·모서리·하단 인셋, L-5 헤더), select-type(M-6 세로 중앙), add-plan 폼(M-1 칩 정렬,
> L-1 `+/−` 라벨·색, M-11 컨디션 원 크기).
> 원복 후에도 `tintText`/`onTint` 색 토큰은 그대로 두어 대비 개선은 유지된다.
> 추가로 FAB 메뉴("루틴 추가"·"운동 추가") 라벨은 60% 검정 오버레이 위에 뜨므로
> 테마와 무관하게 `#fff` 고정으로 바꿨다 — 라이트 모드에서 검은 글자라 읽히지 않았다.

### 색 토큰 3개 신설 (`constants/Colors.ts`)

| 토큰 | light | dark | 용도 |
|---|---|---|---|
| `tintText` | `#00786E` | `#00ccbb` | **글자·아이콘용** teal. 흰 카드 대비 5.4:1 / 앱 배경 대비 4.65:1 |
| `onTint` | `#1a1a1a` | `#1a1a1a` | tint로 **채운 면 위**에 올리는 글자·아이콘 (양 테마 7:1 이상) |
| `divider` | `#D8D7DE` | `#555555` | 카드 구분선. 기존 `tabIconDefault`는 라이트에서 배경색과 동일했다 |

`tabIconDefault`는 원래 용도(탭바 비활성 색)로 되돌려 `#8A8A8E`/`#8E8E93`로 잡았다.

## 6. 확인하지 못한 것

- **카메라 화면**(`app/add-plan/camera.tsx`) — 시뮬레이터에 카메라가 없어 취소 아이콘 변경을

- **`camera-image.tsx` 2열 그리드** — 촬영 경로를 못 타서 미확인. 다만 동일 로직이
  `workout-plan.tsx`에서 정상 동작하는 것을 확인했다.
- **애니메이션 곡선/순서**(FAB 회전, 아코디언 400ms, 계산기 탭 슬라이드) — 정지 프레임으로는
  판정할 수 없다. "요소가 나타난다"까지만 확인했다.
- **숏츠 그리드 하단 잘림** — 영상이 1개뿐이라 여러 줄 상태를 재현하지 못했다. 코드 확인.
- **L-6 정상 저장 경로** — 범위를 벗어난 값이 막히는 것은 시뮬레이터에서 확인했으나

  "정상 값으로 저장 성공"까지는 재현하지 못했다. 검증은 기존 필수값 체크 바로 뒤에
  붙인 `some()` 하나뿐이라 저장 경로 자체는 그대로다.
